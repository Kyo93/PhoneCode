# Technology Stack

**Analysis Date:** 2026-04-07

## Languages

**Primary:**
- JavaScript (ES6+ with ESM) - Frontend client and backend server
- Used throughout: `public/js/app.js`, `server.js`, `targets/*.js`

## Runtime

**Environment:**
- Node.js (v16+)
- See `package.json`: `"engines": {"node": ">=16.0.0"}`

**Package Manager:**
- npm 
- Lockfile: `package-lock.json` (present)

## Frameworks

**Core:**
- Express.js 4.18.2 - REST API server and static file serving
- WebSocket (ws) 8.18.0 - Real-time bidirectional communication between client and desktop

**Chrome DevTools Protocol (CDP):**
- CDP WebSocket connections to local debuggable browser instances (Antigravity on port 9000-9003)
- Custom implementation in `server.js` via WebSocket connections

**Client-Side:**
- Vanilla JavaScript (no framework)
- DOM manipulation and event handling via JavaScript

## Key Dependencies

**Critical:**
- express 4.18.2 - HTTP server framework and routing
- ws 8.18.0 - WebSocket server for real-time updates
- compression 1.8.1 - HTTP response compression middleware
- cookie-parser 1.4.7 - Session cookie handling
- dotenv 16.4.7 - Environment variable configuration

**Infrastructure:**
- None (pure Node.js, no ORM or database)

## Configuration

**Environment:**
- Configuration via `.env` file (not committed)
- Key variables:
  - `APP_PASSWORD` - Password for mobile interface access
  - `NGROK_AUTHTOKEN` - ngrok tunnel authentication
  - `PORT` - Server port (default 3000)
  - `SESSION_SECRET` - Secure session secret
  - `AUTH_SALT` - Salt for auth token hashing
  - Optional: `xxx_API_KEY` - Placeholder for AI provider keys (Groq, Gemini, etc.)

**Build:**
- No build process - files served directly
- Entry point: `server.js`
- Static files in `public/` directory

## Platform Requirements

**Development:**
- Windows, macOS, or Linux
- Node.js >= 16.0.0
- npm package manager
- Local browser with debugging protocol support (Chrome, Edge, Chromium-based)

**Production:**
- Desktop application with Chrome/Chromium debugging port exposed (9000-9003)
- ngrok or similar tunnel for public access (optional)
- SSL certificates (optional, auto-generated via `generate_ssl.js`)

## Key Technical Details

**SSL/TLS Support:**
- Optional HTTPS via self-signed certificates
- Certificates stored in `certs/server.key` and `certs/server.cert`
- Generated by `generate_ssl.js`
- Falls back to HTTP if certificates missing

**Authentication:**
- Password-based with hashed token
- Cookie-based sessions (30-day expiration)
- Local network exemption (same Wi-Fi devices bypass auth)
- Magic link auto-login via `?key=password` query parameter

**Network:**
- HTTP/HTTPS server on configurable port (default 3000)
- WebSocket connections to CDP targets on ports 9000-9003
- Local IP detection for direct network access
- Supports ngrok tunneling for remote access
