<decisions>
## Implementation Decisions

### Auto-resume on Connect
- **D-01:** Only trigger auto-resume when empty state is showing (hasChatOpen returns false / no active conversation) — do NOT override if Claude Code is already displaying a conversation
- **D-02:** "Most recent conversation" = `.jsonl` file with latest `mtime` in `~/.claude/projects/` — already available from `/chat-history` endpoint sorted by mtime

### History Panel Navigation
- **D-03:** CDP click Claude Code's UI to navigate — find and click the correct element inside `#active-frame` iframe that corresponds to the selected session
- **D-04:** Navigation must happen inside `active-frame` iframe context (same pattern as `injectMessage` and `getToolbarState` in `targets/claude.js`)

### New Chat Button
- **D-05:** CDP click the New Chat button inside Claude Code's webview — find via `#active-frame` contentDocument, look for `+` / new conversation button
- **D-06:** Same approach as `startNewChat()` in server.js but searching inside `active-frame` doc instead of top-level document

### Claude's Discretion
- Exact DOM selectors for Claude Code's history panel and new chat button — researcher should inspect actual Claude Code UI structure
- Fallback strategy if CDP click fails (e.g., return error vs silent fail)
