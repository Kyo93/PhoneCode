# Plan 05-04: Gap Closure — effectiveCSS Undefined Guard + CDP Reconnect Cache Invalidation

**Phase**: 5 — Capture Pipeline Optimization
**Goal**: Close two related gaps that compound to produce broken CSS after server restart when a browser tab remains open.

---

## Gap Summary

Two bugs interact to produce a server-restart regression:

**Gap 2 (root cause):** `initCDP()` calls `cdpConnections.set()` but never calls `invalidateSnapshotCache()` on the freshly connected target. After a server restart, the browser retains all 9 cache globals from the previous session — including `__phoneCodeObserver` and `__phoneCodeLastHTML`. The MutationObserver early-return fires on the very first poll of the reconnected session, returning `{ html: cachedHtml, ...lastMeta }` with no `css` field (`snapshot.css === undefined`).

**Gap 1 (mechanism):** The `effectiveCSS` guard at server.js line 1412 uses strict `!== null`. `undefined !== null` evaluates to `true`, so `effectiveCSS = undefined`. `lastSnapshot.css = undefined`. `GET /snapshot` returns `{ css: undefined }` to new clients — styles not injected, visual regression.

**Gap 2 fix prevents Gap 1 from manifesting** (invalidated globals force full capture). **Gap 1 fix is defense-in-depth** for any future code path that omits the `css` field.
