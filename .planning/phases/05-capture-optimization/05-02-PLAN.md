# Plan 05-02: CSS Collection Cache

**Phase**: 5 — Capture Pipeline Optimization
**Goal**: Skip CSS re-collection when stylesheets haven't changed since the last poll.

---

## Problem

Both targets collect **all** CSS rules on every snapshot:
```js
// claude.js lines 88–104, antigravity.js lines 148–156
const rules = [];
for (const sheet of document.styleSheets) {
    try {
        for (const rule of sheet.cssRules) {
            rules.push(rule.cssText);
        }
    } catch(e) {}
}
const allCSS = rules.join('\n');
```
For Claude Code / VS Code: thousands of rules, 100KB+ of CSS text — every second, regardless of whether any stylesheet changed.

**Note**: Phase 4 already added a server-side CSS hash guard (`lastBroadcastCssHash`) — the CSS is not *broadcast* when unchanged. But it is still **collected in the browser and returned via CDP on every poll** — wasting browser CPU and CDP transfer bandwidth.

## Solution

Add `window.__phoneCodeCSSFingerprint` — a lightweight fingerprint (stylesheet count + per-sheet rule count + first-rule sample) checked before re-collecting. If fingerprint matches, return `null` for CSS. Server handles `css: null` via the fix in Task 1.

---

## Pre-conditions

- Plan 05-01 complete: `invalidateSnapshotCache()` already clears `window.__phoneCodeCSSFingerprint` (the CLEAR_SCRIPT includes it)
- `server.js` polling loop contains `lastBroadcastCssHash` CSS logic from Phase 4

## Pre-existing Escaping Bugfix (separate from caching change)

**Bug**: In the current `targets/claude.js` CAPTURE_SCRIPT, the CSS transform regexes inside the template literal use single backslash (`\s`, `\d`). Inside a JavaScript template literal string, `\s` becomes `s` — the regex receives `s` instead of the whitespace class. This is a pre-existing bug that causes the transforms to silently fail (no error, just no transformation).

**Fix**: Tasks 2 and 3 use double-backslash (`\\s`, `\\d`) throughout the regex replacements. This is **not** a caching change — it fixes a pre-existing correctness bug in `claude.js` that has existed since the transforms were added. `antigravity.js` does not apply these transforms (it uses raw `rule.cssText`), so no escaping fix is needed there.

Do NOT revert to single-backslash when copying code from the current file.

---

## Critical Task Ordering

**Tasks 2 and 3 make `allCSS` potentially `null`**. The existing `stats.cssSize: allCSS.length` will throw `TypeError: Cannot read properties of null` on a cache hit until Task 4 adds the null guard. **Apply Tasks 2, 3, and 4 together** — do not start the server between Task 3 and Task 4.

**Task 1 (server null-CSS guard) should be deployed before Tasks 2–4 go live** in production. In development (single server instance), Task 1 can be applied in the same edit session.

## Cross-origin Sheet Behavior

For external sheets (`s.href` exists) where `s.cssRules` throws a `SecurityError` (cross-origin CORS restriction):

- The outer `try/catch` catches the error and returns `'|blocked:' + i` (stable by sheet index)
- On subsequent polls with the same sheets at the same indices, the fingerprint entry is identical → cache hit for those sheets
- This is **correct behavior**: cross-origin sheets cannot contribute to `allCSS` regardless (the collection loop also throws for them), so their CSS contribution is always empty
- Cache hits for blocked sheets produce identical output to cache misses — no correctness impact
- This trade-off is documented in the code comment on the catch block

**With:**
```js
// CSS fingerprint — detects stylesheet add/remove and content changes.
// External sheets: href + ruleCount + first-rule sample (64 chars).
//   ruleCount alone misses in-place value changes (CSS variables, color tweaks) without
//   adding or removing rules. First-rule sample provides cheap collision discriminator.
// Inline <style>: hash ownerNode.textContent to catch any content change.
// Cross-origin sheets (SecurityError on cssRules): caught below → '|blocked:N' (stable
//   by sheet index). Acceptable: cross-origin rules can't be collected either, so their
//   CSS contribution is always empty regardless of fingerprint outcome.
const cssFingerprint = Array.from(document.styleSheets).reduce((acc, s, i) => {
    try {
        if (s.href) {
            // External stylesheet: href + rule count + first-rule content sample.
            const firstRule = s.cssRules.length > 0 ? s.cssRules[0].cssText.slice(0, 64) : '';
            return acc + '|' + s.href + ':' + s.cssRules.length + ':' + firstRule;
        } else {
            // Inline <style>: hash content to avoid collision when rule counts match.
            const content = s.ownerNode?.textContent || '';
            let h = 0;
            for (let j = 0; j < content.length; j++) {
                h = (Math.imul(31, h) + content.charCodeAt(j)) | 0;
            }
            return acc + '|inline' + i + ':' + h;
        }
    } catch(e) {
        // Cross-origin sheet or other access error — stable index-based fingerprint.
        // CSS collection also skips these sheets (same SecurityError), so output is unaffected.
        return acc + '|blocked:' + i;
    }
}, '');

let allCSS;
if (cssFingerprint === window.__phoneCodeCSSFingerprint && window.__phoneCodeCSSCache) {
    // Stylesheets unchanged — signal cache hit by returning null. Server keeps prior CSS.
    allCSS = null;
} else {
    // Fingerprint changed (or first poll) — collect all CSS rules.
    const rules = [];
    for (const sheet of document.styleSheets) {
        try {
            for (const rule of sheet.cssRules) {
                let text = rule.cssText;
                // NOTE: Use \\s and \\d (double-backslash) inside template literal.
                // Single \s becomes literal 's' when the string is evaluated — regex broken.
                // This also fixes a pre-existing escaping bug in the original claude.js code.
                text = text
                    .replace(/position\s*:\s*fixed/gi, 'position: relative')
                    .replace(/position\s*:\s*sticky/gi, 'position: relative')
                    .replace(/z-index\s*:\s*\d{3,}/gi, 'z-index: 1')
                    .replace(/height\s*:\s*100vh/gi, 'height: auto')
                    .replace(/min-height\s*:\s*100vh/gi, 'min-height: 0');
                rules.push(text);
            }
        } catch(e) {}
    }
    allCSS = rules.join('\\n');
    window.__phoneCodeCSSFingerprint = cssFingerprint;
    window.__phoneCodeCSSCache = allCSS;
}
```

**Important**: The regex strings in the collection loop use `\\s` and `\\d` (double-backslash). Inside a template literal, `\s` becomes `s` — the regex engine receives `s` instead of the whitespace class, silently breaking the transforms. The original `claude.js` has this bug on lines ~95–100. Do NOT revert to single-backslash.

The `allCSS = null` on cache hit is intentional. The server Task 1 fix ensures `null` is normalized to the prior effective CSS before reaching `lastSnapshot` or the broadcast — the client always receives valid styles.

### Null CSS server contract test
1. When target returns `css: null`:
   - Server should NOT update `lastBroadcastCssHash`
   - Server should NOT include `css` in `snapshot_diff` broadcast
   - `lastSnapshot.css` must remain concrete text (not null)
   - Client reconnect via `GET /snapshot` should return real CSS, not null

### Strengthened fingerprint test
1. Simulate CSS variable change in browser devtools (same rule count, different value)
2. On next poll: fingerprint changes (firstRule text differs) → `cssCached: false` → CSS re-collected
